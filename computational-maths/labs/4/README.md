# Численное интегрирование методом трапеций

Реализация алгоритма вычисления определённого интеграла от заданной функции на интервале $[a, b]$ с использованием метода трапеций. Алгоритм обеспечивает заданную точность $\varepsilon$ путём автоматического увеличения количества разбиений отрезка интегрирования.

## Алгоритм работы

1.  **Подготовка данных и выбор функции**:
    Исходные параметры считываются из входного потока: границы интервала $[a, b]$, номер функции $f$ для выбора подынтегральной функции, и параметр точности $\varepsilon$.
2.  **Диагностика особенностей функции**:
    Перед вычислением проводится анализ подынтегральной функции на наличие разрывов и точек неопределённости на интервале $[a, b]$. При обнаружении разрыва второго рода, скачка или неопределённости функции устанавливаются флаги `error_message` и `hasDiscontinuity`, выводится соответствующее сообщение.
3.  **Итерационный процесс интегрирования**:
    Вычисление интеграла методом трапеций производится по формуле:
    $$I \approx \frac{h}{2} \left[ f(a) + 2\sum_{i=1}^{n-1} f(x_i) + f(b) \right]$$
    где $h = \frac{b-a}{n}$ — шаг разбиения, $n$ — количество отрезков.
    Процесс начинается с минимального количества разбиений и последовательно удваивается до достижения заданной точности.
4.  **Контроль точности**:
    На каждой итерации вычисляется разница между текущим и предыдущим приближениями интеграла. Процесс продолжается до тех пор, пока $|I_{n} - I_{n/2}| \leq \varepsilon$.
5.  **Обработка особых случаев**:
    *   При наличии устранимого разрыва первого рода вычисление выполняется корректно.
    *   Если $a > b$, результат вычислений умножается на $-1$.

## Формат ввода
Программа ожидает строку, содержащую границы интервала, номер функции и параметр точности:
`a b f epsilon`

* **a**: Левая граница интервала интегрирования.
* **b**: Правая граница интервала интегрирования.
* **f**: Порядковый номер подынтегральной функции.
* **epsilon**: Параметр точности $\varepsilon$ — максимальная допустимая разница между двумя последовательными итерациями.

## Формат вывода
Программа выводит вычисленное значение определённого интеграла:
`I`

> **Примечание**: Если функция имеет разрыв второго рода или не определена на интервале $[a, b]$, выводится сообщение: *"Integrated function has discontinuity or does not defined in current interval"*. При $a > b$ значение интеграла будет отрицательным. Точность результата гарантируется в пределах заданного параметра $\varepsilon$.
